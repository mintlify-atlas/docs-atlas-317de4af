title: "Quickstart"
description: "Get SIUT-ITSM running locally in under five minutes."
---

This guide walks you through cloning the repository, installing dependencies, and starting the built-in PHP development server so you can explore the platform immediately.

<Steps>
  <Step title="Clone the repository">
    Clone the project from GitHub and enter the directory:

    ```bash
    git clone https://github.com/mrtnsnchz/ost-siut-itsm.git
    cd ost-siut-itsm
    ```
  </Step>

  <Step title="Install PHP dependencies">
    Install all Composer dependencies (Latte, PHPMailer, PHP-DI, Monolog, dompdf, and more):

    ```bash
    composer install
    ```

    <Note>
      Composer 2.x and PHP 8.3 or higher are required. Run `php -v` and `composer -V` to confirm your versions before proceeding.
    </Note>
  </Step>

  <Step title="Copy the environment file">
    Copy the example environment file and open it for editing:

    ```bash
    cp .env.example .env
    ```

    At minimum, update the database credentials to match your local MySQL or MariaDB instance:

    ```env
    DB_HOST=localhost
    DB_NAME=siut_itsm
    DB_USER=your_db_user
    DB_PASSWORD=your_db_password
    DB_PORT=3306
    ```

    You can leave the mailer settings as placeholders for now if you only want to explore the interface.
  </Step>

  <Step title="Create the database and import the schema">
    Create a database in MySQL or MariaDB, then import the schema:

    ```bash
    mysql -u your_db_user -p -e "CREATE DATABASE IF NOT EXISTS siut_itsm CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
    mysql -u your_db_user -p siut_itsm < database.sql
    ```

    To also load sample data for testing:

    ```bash
    mysql -u your_db_user -p siut_itsm < database.seed.sql
    ```
  </Step>

  <Step title="Start the development server">
    Use the Composer script to launch PHP's built-in web server:

    ```bash
    composer run servidor
    ```

    This runs `php -S localhost:8888 -t .` and serves the application from the project root.
  </Step>
</Steps>

You're now running SIUT-ITSM at **http://localhost:8888**.

<Tip>
  The `APP_ENV=dev` value in your `.env` file enables development mode, which disables secure cookie settings and makes debugging easier. Never use `APP_ENV=dev` in a production environment.
</Tip>
