## Overview

SIUT-ITSM uses MySQL with the `utf8mb4` character set throughout. The schema is defined in `database.sql` at the project root. To initialise a fresh database:

```bash
mysql -u root -p your_database < database.sql
```

A seed file with sample data is also available:

```bash
mysql -u root -p your_database < database.seed.sql
```

The PDO connection is configured in `config/definitions.php` using `PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION` and `PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC`.

<Accordion title="users">

Core user table. Stores authentication credentials, personal information, banking data, and employment details.

```sql
CREATE TABLE IF NOT EXISTS users (
    user_id         INT AUTO_INCREMENT PRIMARY KEY,
    -- auth
    email           VARCHAR(100)   NOT NULL UNIQUE,
    password_hash   VARCHAR(255)   NOT NULL,
    role            VARCHAR(20)    NOT NULL DEFAULT 'no_agremiado',
    active          BOOLEAN                 DEFAULT TRUE,
    -- personal
    curp            VARCHAR(20),
    name            VARCHAR(100)   NOT NULL,
    surnames        VARCHAR(255)   NOT NULL,
    birthdate       DATE                    DEFAULT NULL,
    address         VARCHAR(255),
    phone           VARCHAR(50),
    photo           VARCHAR(255),
    -- banking
    bank_name       VARCHAR(100),
    interbank_code  VARCHAR(18),
    bank_account    VARCHAR(20),
    -- employment
    category        VARCHAR(100),
    department      VARCHAR(100),
    nss             VARCHAR(15),
    salary          DECIMAL(12,2)           DEFAULT 0,
    work_start_date DATE                    DEFAULT NULL,
    -- session
    last_entry      DATETIME                DEFAULT NULL,
    created_at      DATETIME                DEFAULT CURRENT_TIMESTAMP,
    update_at       DATETIME                DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    delete_at       DATETIME                DEFAULT NULL
);
```

| Column | Type | Notes |
|---|---|---|
| `user_id` | INT | Auto-increment primary key |
| `email` | VARCHAR(100) | Unique; used as login identifier |
| `password_hash` | VARCHAR(255) | Bcrypt hash |
| `role` | VARCHAR(20) | `admin`, `lider`, `agremiado`, `no_agremiado` |
| `active` | BOOLEAN | Soft-disable flag |
| `curp` | VARCHAR(20) | Mexican CURP identifier |
| `nss` | VARCHAR(15) | IMSS social security number |
| `salary` | DECIMAL(12,2) | Base monthly salary |
| `delete_at` | DATETIME | Soft-delete timestamp |

</Accordion>

<Accordion title="loans">

Main loan record. Tracks the full lifecycle from application through disbursement to full liquidation.

```sql
CREATE TABLE IF NOT EXISTS loans (
    loan_id                     INT AUTO_INCREMENT PRIMARY KEY,
    user_id                     INT            NOT NULL,
    folio                       VARCHAR(50)    UNIQUE,
    requested_amount            DECIMAL(10,2)  NOT NULL,
    approved_amount             DECIMAL(10,2),
    applied_interest_rate       DECIMAL(5,2)   NOT NULL,
    daily_default_rate          DECIMAL(5,4),
    estimated_total_to_pay      DECIMAL(10,2),
    outstanding_balance         DECIMAL(10,2),
    term_months                 INT,
    term_fortnights             INT,
    first_payment_date          DATE,
    last_scheduled_payment_date DATE,
    -- Workflow timestamps
    application_date            DATETIME DEFAULT CURRENT_TIMESTAMP,
    document_review_date        DATETIME,
    approval_date               DATETIME,
    document_generation_date    DATETIME,
    signature_validation_date   DATETIME,
    disbursement_date           DATETIME,
    total_liquidation_date      DATETIME,
    status                      VARCHAR(30) NOT NULL DEFAULT 'borrador',
    original_loan_id            INT,
    rejection_reason            TEXT,
    admin_observations          TEXT,
    internal_observations       TEXT,
    finance_signatory           VARCHAR(255),
    lender_signatory            VARCHAR(255),
    requires_restructuring      BOOLEAN DEFAULT FALSE,
    created_by                  INT,
    deletion_date               DATETIME DEFAULT NULL,
    CONSTRAINT fk_loan_user   FOREIGN KEY (user_id)         REFERENCES users (user_id) ON DELETE RESTRICT,
    CONSTRAINT fk_loan_origin FOREIGN KEY (original_loan_id) REFERENCES loans (loan_id) ON DELETE SET NULL,
    INDEX idx_folio            (folio),
    INDEX idx_user_status      (user_id, status),
    INDEX idx_status_date      (status, application_date),
    INDEX idx_origin           (original_loan_id)
);
```

| Column | Type | Notes |
|---|---|---|
| `folio` | VARCHAR(50) | Auto-generated folio, e.g. `SIN-2025-001` |
| `applied_interest_rate` | DECIMAL(5,2) | Exact percentage applied (can be customised) |
| `daily_default_rate` | DECIMAL(5,4) | Used to calculate late payment penalties |
| `outstanding_balance` | DECIMAL(10,2) | Updated after each payment |
| `status` | VARCHAR(30) | `borrador`, `revision_documentos`, `aprobado`, `generacion_documentos`, `validacion_firma`, `desembolsado`, `liquidado`, `rechazado` |
| `original_loan_id` | INT | Self-referencing FK; set when this loan is a restructuring |
| `disbursement_date` | DATETIME | Start of interest accrual |

</Accordion>

<Accordion title="loan_amortization">

Amortisation schedule rows — one row per scheduled payment instalment.

```sql
CREATE TABLE IF NOT EXISTS loan_amortization (
    amortization_id            INT AUTO_INCREMENT PRIMARY KEY,
    loan_id                    INT            NOT NULL,
    payment_number             INT            NOT NULL,
    income_type_id             INT            NOT NULL,
    scheduled_date             DATE           NOT NULL,
    initial_balance            DECIMAL(10,2)  NOT NULL,
    principal                  DECIMAL(10,2)  NOT NULL,
    ordinary_interest          DECIMAL(10,2)  NOT NULL,
    total_scheduled_payment    DECIMAL(10,2)  NOT NULL,
    final_balance              DECIMAL(10,2)  NOT NULL,
    payment_status             VARCHAR(30)    NOT NULL DEFAULT 'pendiente',
    actual_payment_date        DATETIME,
    actual_paid_amount         DECIMAL(10,2)           DEFAULT 0,
    days_overdue               INT                     DEFAULT 0,
    generated_default_interest DECIMAL(10,2)           DEFAULT 0,
    paid_by                    INT,
    payment_receipt            VARCHAR(255),
    table_version              INT                     DEFAULT 1,
    active                     BOOLEAN                 DEFAULT TRUE,
    CONSTRAINT fk_amort_loan        FOREIGN KEY (loan_id)       REFERENCES loans (loan_id)           ON DELETE CASCADE,
    CONSTRAINT fk_amort_income_type FOREIGN KEY (income_type_id) REFERENCES cat_income_types (income_type_id) ON DELETE RESTRICT,
    INDEX idx_loan_number   (loan_id, payment_number),
    INDEX idx_date_status   (scheduled_date, payment_status),
    INDEX idx_version_active (loan_id, table_version, active)
);
```

| Column | Type | Notes |
|---|---|---|
| `payment_number` | INT | Sequence number within the loan (1 … N) |
| `scheduled_date` | DATE | 15th or 20th of each month |
| `principal` | DECIMAL(10,2) | Capital portion of this instalment |
| `ordinary_interest` | DECIMAL(10,2) | Interest portion |
| `generated_default_interest` | DECIMAL(10,2) | Late-payment penalty accumulated |
| `table_version` | INT | Incremented when the schedule is regenerated |
| `active` | BOOLEAN | `FALSE` for rows from superseded schedule versions |

</Accordion>

<Accordion title="mail_queue">

Persistent outbound mail queue. Messages are inserted here synchronously and delivered asynchronously by `scripts/mail-queue.php`.

```sql
CREATE TABLE IF NOT EXISTS mail_queue (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    recipient    TEXT         NOT NULL,
    subject      VARCHAR(255) NOT NULL,
    body         MEDIUMTEXT   NOT NULL,
    alt_body     TEXT,
    priority     TINYINT                  DEFAULT 2,
    attempts     INT                      DEFAULT 0,
    max_attempts INT                      DEFAULT 3,
    status       ENUM('pending','sending','sent','failed') DEFAULT 'pending',
    last_error   TEXT,
    scheduled_at TIMESTAMP                DEFAULT CURRENT_TIMESTAMP,
    locked_at    TIMESTAMP NULL,
    lock_token   VARCHAR(64) NULL,
    created_at   TIMESTAMP                DEFAULT CURRENT_TIMESTAMP,
    INDEX (status, priority, scheduled_at),
    INDEX (status, lock_token)
);
```

| Column | Type | Notes |
|---|---|---|
| `priority` | TINYINT | `1` = Urgent, `2` = Normal, `3` = Newsletter |
| `attempts` | INT | How many delivery attempts have been made |
| `max_attempts` | INT | Default `3`; messages exceeding this are marked `failed` |
| `status` | ENUM | `pending` → `sending` → `sent` or `failed` |
| `lock_token` | VARCHAR(64) | Process identifier; prevents two cron workers from sending the same message |
| `locked_at` | TIMESTAMP | When the lock was acquired |

</Accordion>

<Accordion title="localidades (created at runtime)">

This table is **not** in `database.sql` — it is created on the fly by `commands/importar-direcciones.php` when you run the address import.

```sql
CREATE TABLE IF NOT EXISTS localidades (
    id        INT AUTO_INCREMENT PRIMARY KEY,
    cp        VARCHAR(10)  NOT NULL,
    colonia   VARCHAR(255) NOT NULL,
    localidad VARCHAR(255) NOT NULL,
    ciudad    VARCHAR(255) NOT NULL,
    estado    VARCHAR(255) NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

ALTER TABLE localidades ADD INDEX idx_cp (cp);
```

Data is sourced from the official SEPOMEX postal code file (`CPdescarga.txt` inside a ZIP). All text is normalised to uppercase ASCII via `Transliterator::create("Any-Latin; Latin-ASCII; Upper()")` before insertion.
